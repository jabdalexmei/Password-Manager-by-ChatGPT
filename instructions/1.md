

## TS (EN): Fix `tauri-plugin-opener` open_path PathBuf type error

### Goal

Fix compilation error:
`E0277: the trait bound String: From<PathBuf> is not satisfied`
in `src-tauri/src/services/attachments_service.rs`.

### File

`src-tauri/src/services/attachments_service.rs`

### Change

Locate:

```rust
app.opener()
    .open_path(preview_path, None)
    .map_err(|_| ErrorCodeString::new("ATTACHMENT_OPEN_FAILED"))
```

Replace with:

```rust
let preview_path_str = preview_path.to_string_lossy().into_owned();

app.opener()
    .open_path(preview_path_str, None)
    .map_err(|_| ErrorCodeString::new("ATTACHMENT_OPEN_FAILED"))
```

### Acceptance criteria

* `cargo build` succeeds past this error.
* “Open attachment” continues to open the decrypted preview file via the OS default app.
