## TS: Fix toast layout (make `.toast` a flex-row) and correct close button position

### Goal

* Toast notification card must be a **single horizontal row**: **text/content on the left**, **close (X) button on the right**.
* Close button must be vertically aligned nicely (top/center — выбери одно, ниже зафиксировал *top* как обычно удобнее).
* No absolute positioning for the close button.

### Where to change in the project

#### 1) Toast styles

**File:** `src/ui.css`
Current toast styles are defined here (host + toast card). 

#### 2) Toast component (render + close button)

**File:** `src/components/Toaster.tsx` (or `src/components/Toaster/index.tsx` depending on your structure)
This is confirmed by imports like: `useToaster` from `../../../../components/Toaster`.

## Implementation details

### A) CSS changes (make toast a flex row + close button pinned right)

**File:** `src/ui.css`

#### A1. Update `.toast` to be a row container

Replace/extend the existing `.toast { ... }` block (it currently exists). 

**New required CSS (exact):**

```css
/* Toast becomes a row: content left, close button right */
.toast {
  min-width: 240px;
  background: var(--color-surface-2);
  border: 1px solid var(--color-border);
  border-radius: var(--radius-sm);
  padding: 12px 14px;
  color: var(--color-text);
  box-shadow: var(--shadow-card);

  display: flex;
  flex-direction: row;
  align-items: flex-start; /* keep X aligned with title area */
  gap: 12px;
}

/* Toast text/content column */
.toast-body {
  display: flex;
  flex-direction: column;
  gap: 4px;
  min-width: 0;
  flex: 1 1 auto;
}

/* Close button pinned to the right */
.toast-close {
  margin-left: auto;
  flex: 0 0 auto;
  align-self: flex-start;

  width: 28px;
  height: 28px;
  padding: 0;

  display: inline-flex;
  align-items: center;
  justify-content: center;

  border: 1px solid transparent;
  background: transparent;
  color: var(--color-text-muted);
  border-radius: var(--radius-sm);
  cursor: pointer;
}

.toast-close:hover {
  border-color: var(--color-border);
  color: var(--color-text);
}
```

> Почему так: flex + `margin-left:auto` — стандартный способ “вытолкнуть” последний элемент вправо в flex-row. ([MDN Web Docs][1])

### B) Toaster component changes (wrap content + render close button consistently)

**File:** `src/components/Toaster.tsx` (или эквивалентный файл, который рендерит `.toast-host` и `.toast`)

#### B1. Required markup structure per toast

For each toast item, ensure this exact structure:

```tsx
<div className={`toast ${variant === 'success' ? 'toast-success' : ''} ${variant === 'error' ? 'toast-error' : ''}`}>
  <div className="toast-body">
    <p className="toast-title">{title}</p>
    {description ? <p className="toast-description">{description}</p> : null}
  </div>

  <button
    type="button"
    className="toast-close"
    aria-label="Close"
    onClick={() => dismissToast(id)}
  >
    ×
  </button>
</div>
```

**Rules:**

* Close button must be the **last child** inside `.toast` so that `margin-left: auto` works.
* Do **not** absolutely-position the close button.
* Keep `.toast-title` / `.toast-description` classes as-is (they already exist in `ui.css`). 
* `aria-label="Close"` is required for accessibility (close icon alone is not readable to screen readers). ([Carbon Design System][2])

*(Если у тебя close сейчас рисуется иконкой — ок, но тогда иконке поставь `aria-hidden="true"`, а label оставь на button.)* ([Carbon Design System][2])

## Acceptance criteria

1. Toast выглядит как **одна строка**: текст слева, крестик строго справа.
2. Крестик не “уезжает” и не перекрывает текст при длинных сообщениях (текст должен аккуратно обрезаться/переноситься внутри `.toast-body`).
3. Close button кликабельный, имеет `aria-label="Close"`. ([Carbon Design System][2])
4. `.toast-host` позиционирование не меняем (у тебя уже top-right и column).
