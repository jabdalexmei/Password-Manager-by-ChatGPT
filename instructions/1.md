
# Technical Specification (EN): Fix swapped Import/Export icons in Vault header

## Goal

Ensure the **Export** and **Import** toolbar icons match the actions they trigger:

* Export button must visually show the **Export** icon.
* Import button must visually show the **Import** icon.
* Actions (onClick handlers) must remain correct:

  * Export → opens export flow/modal
  * Import → opens import flow/file picker

## Scope

Frontend only. No backend changes.

## Files

* `src/features/Vault/components/Header/VaultHeader.tsx`

## Implementation

### 1) Swap icon components rendered for Export and Import buttons

**File:** `src/features/Vault/components/Header/VaultHeader.tsx`

Locate this block:

```tsx
<button className="vault-action-button" type="button" aria-label={t('export')} onClick={onExportBackup}>
  <IconDownload />
</button>
<button className="vault-action-button" type="button" aria-label={t('import')} onClick={onImportBackup}>
  <IconImport />
</button>
```

Replace it with:

```tsx
<button className="vault-action-button" type="button" aria-label={t('export')} onClick={onExportBackup}>
  <IconImport />
</button>
<button className="vault-action-button" type="button" aria-label={t('import')} onClick={onImportBackup}>
  <IconDownload />
</button>
```

**Do not change** the `onClick` handlers or `aria-label` keys—only swap the rendered icons.

## Acceptance Criteria

1. Clicking the icon that looks like **Export** triggers **export backup** UI/flow.
2. Clicking the icon that looks like **Import** triggers **import/restore** UI/flow.
3. `aria-label`/tooltips remain correct and consistent with the action.
4. No other toolbar icons are affected.

## Notes

* Your icon wrappers currently map Lucide icons to project names (Download/Upload as React components). Lucide provides separate “download” and “upload” icons. ([Lucide][1])
