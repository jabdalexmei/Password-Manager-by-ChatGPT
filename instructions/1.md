

# TS (EN): Swap backup import/export icons to match arrow semantics (Down=Import, Up=Export) + keep actions unchanged

## Problem

In the Vault header action bar, the **Export action** is currently rendered with a **down-arrow icon**, and the **Import action** is rendered with an **up-arrow icon**. This is visually misleading.

## Expected

* **Import** must show **down arrow** (download/down).
* **Export** must show **up arrow** (upload/up).
* **onClick handlers must NOT change**: Export still triggers export flow; Import still triggers restore/import flow.
* Tooltips must match the action.

## Files

* `src/components/lucide/icons.ts`
* `src/features/Vault/components/Header/VaultHeader.tsx`
* (Optional, if you use tooltip namespace) `src/i18n/English/Tooltips.json`

---

## Change 1: Fix icon semantic exports in `icons.ts`

### File: `src/components/lucide/icons.ts`

#### Current (wrong semantics)

You likely have something like:

```ts
export { Download as IconDownload, Upload as IconImport } from 'lucide-react';
```

#### Required (explicit semantic names)

Replace/introduce **semantic** exports:

```diff
diff --git a/src/components/lucide/icons.ts b/src/components/lucide/icons.ts
--- a/src/components/lucide/icons.ts
+++ b/src/components/lucide/icons.ts
@@
-export { Download as IconDownload, Upload as IconImport } from 'lucide-react';
+// Down arrow = Import (download into the app)
+export { Download as IconImport } from 'lucide-react';
+// Up arrow = Export (upload / send out of the app)
+export { Upload as IconExport } from 'lucide-react';
```

> Почему так: “download/down” обычно воспринимается как получение внутрь, “upload/up” — отправка наружу; из-за контекста import/export бывает спорным, но ты хочешь именно такую семантику. ([User Experience Stack Exchange][1])
