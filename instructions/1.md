## Technical Specification (TS) — Remove vendored `lucide-react` override and use npm dependency

### Title

Remove TypeScript path override for `lucide-react` and delete vendored copy

### Goal

* Stop overriding imports of `lucide-react` to `./vendor/lucide-react`
* Use the official `lucide-react` dependency installed from npm (`node_modules`)
* Remove unused vendored directory to reduce repo noise and avoid TS/runtime resolution divergence

### Background

TypeScript `compilerOptions.paths` remaps module specifiers to custom lookup locations. ([typescriptlang.org][1])
Vite does not automatically resolve `tsconfig paths` unless configured (alias/plugin). ([GitHub][2])
Keeping a TS-only override risks “TS sees vendor, runtime uses node_modules”.

---

### Scope

* **Frontend only**
* No functional UI changes
* No icon renames
* Keep `src/components/lucide/*` as the project-level icon wrapper (do not remove)

---

### Changes

#### 1) Remove the `lucide-react` path mapping from `tsconfig.json`

**File:** `tsconfig.json`

**Current:**

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "lucide-react": ["./vendor/lucide-react"]
    }
  }
}
```

**Required:**

* Remove the `lucide-react` mapping entry from `compilerOptions.paths`.
* Keep other path aliases (e.g. `@/*`) intact if present.

Result example:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      // keep other aliases here (if any)
    }
  }
}
```

> Note: If `"paths"` becomes empty, it can be removed entirely to keep config clean.

**Rationale:** `paths` is designed to remap imports like `"lucide-react"` to custom vendor directories. Removing it restores standard node resolution. ([typescriptlang.org][1])

---

#### 2) Delete vendored directory

**Path to delete:**
`vendor/lucide-react/`

Delete the directory fully from the repo/workspace.

---

#### 3) Ensure dependency is installed from npm

**File:** `package.json`

Verify dependency exists:

```json
"dependencies": {
  "lucide-react": "..."
}
```

Reference: official install and usage. ([npmjs.com][3])

---

#### 4) Fix lockfile if it references the vendor path (important)

If your `package-lock.json` (or pnpm-lock/yarn.lock) previously pinned `lucide-react` to a local path, installs may keep pointing to vendor.

**Required check:**

* Search in `package-lock.json` for `vendor/lucide-react` or `"resolved": "vendor/lucide-react"`.

**If found, required remediation:**

* Delete `node_modules/`
* Delete `package-lock.json`
* Run `npm install` to regenerate lockfile and install `lucide-react` from registry

**Rationale:** reproducible installs depend on lockfile; if lock points to non-existent local path, installs break or behave unexpectedly. ([Stack Overflow][4])

---

#### 5) Keep `src/components/lucide` (do not delete)

**Path:** `src/components/lucide/*`

This folder should remain as the internal icon abstraction layer. No changes required unless it imports vendor directly (see next step).

---

#### 6) Verify there are no direct imports from the vendor path

Run project-wide search for:

* `vendor/lucide-react`
* relative imports into vendor (e.g. `../../vendor/lucide-react`)

**Expected:** 0 results after deleting vendor folder.

If any are found, replace them with standard:

```ts
import { IconName } from "lucide-react";
```

(or better: import from your wrapper in `src/components/lucide/icons.ts` if that’s the project convention).

---

### Acceptance Criteria

1. `tsconfig.json` contains **no** `compilerOptions.paths` mapping for `lucide-react`. ([typescriptlang.org][1])
2. Folder `vendor/lucide-react` is removed.
3. `npm install` completes successfully and installs `lucide-react` from registry (`node_modules/lucide-react` exists). ([npmjs.com][3])
4. App builds and runs (`npm run dev` / `npm run build`) with no TS errors “Cannot find module 'lucide-react'”.
5. No runtime differences in icon rendering (same icons appear).

---

### Manual Test Plan

1. Clean install: remove `node_modules`, (and lockfile if it references vendor), run `npm install`. ([Stack Overflow][4])
2. Run `npm run dev` → verify UI loads and icons render.
3. Run `npm run build` → build succeeds.
4. Grep check: no references to `vendor/lucide-react` remain.
