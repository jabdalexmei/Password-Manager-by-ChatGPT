

## TS: Make `ProfileCreate` + `LogIn` screens match `Startup` (Select profile) card size/layout

### Goal

Make **Create profile** and **Log in** screens visually consistent with **Select profile (Startup)**:

* Same outer shell spacing and background.
* Same main card size (`width: min(980px, 100%)`) and padding.
* Content inside card should not look stretched: keep form content centered with a max width.
* Add a consistent footer row: **Back** on the left, **Create / Log in** on the right.

* `src/features/Startup/Startup.tsx` (reference)
* `src/shared/styles/styles/screens/startup.css` (reference)

## Acceptance criteria (must pass)

1. `Startup` card size stays unchanged (reference: `startup.css`).
2. `LogIn` card and `ProfileCreate` card have the same width rule as Startup:

   * `width: min(980px, 100%)`
   * same padding: `32px 36px 28px` (and `24px` on <=720px).
3. In `LogIn` and `ProfileCreate`, form content is centered and not stretched:

   * uses `.auth-content { width: min(640px, 100%); margin: 0 auto; }`
4. Buttons layout:

   * Back is on the left.
   * Create/Log in is on the right.
5. No unused CSS classes remain referenced in these screens (we removed `.login-actions` / `.profile-create-actions` usage).
