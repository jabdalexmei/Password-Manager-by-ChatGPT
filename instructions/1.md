

## Technical Specification (EN): Fix hover border clipping on the first datacard in “All items” list

### Issue

In the Vault “All items” list, the first visible datacard loses part of its border/outline when hovered. This happens only for the top item because the hover animation moves it upward (`translateY(-1px)`) and the scroll container clips the top edge.

### Root cause

* `src/styles/screens/vault.css` contains:

  * `.vault-datacard-list { overflow: auto; ... }` (scroll container)
  * `.vault-datacard:hover { transform: translateY(-1px); ... }`
* When the first item is at the top boundary of the scroll container, moving it up causes its border/outline to be clipped by the container’s overflow.

### Fix strategy

Keep the hover lift effect, but prevent clipping by adding a small inner padding to the scroll container so the first visible item has enough space above it (also prevents `:focus-visible` outline clipping).

---

### Files

* `src/styles/screens/vault.css`

### Change 1: Add inner padding to `.vault-datacard-list`

**Find current rule** (may already be modified in your branch to remove stroke/panel look):

```css
.vault-datacard-list {
  /* background: transparent; */
  /* border: none; */
  /* border-radius: 0; */
  padding: 0;
  overflow: auto;
  flex: 1;
  display: flex;
  flex-direction: column;
  gap: 8px;
}
```

**Update to:**

```css
.vault-datacard-list {
  /* keep the panel stroke removed */
  background: transparent;
  border: none;
  border-radius: 0;

  /* prevent clipping of hover transform + focus-visible outline */
  padding: 4px;

  overflow: auto;
  flex: 1;
  display: flex;
  flex-direction: column;
  gap: 8px;
}
```

**Notes**

* `padding: 4px` is intentionally > 2px to also accommodate `.vault-datacard:focus-visible { outline-offset: 2px; }` without being clipped.
* This does **not** reintroduce the outer stroke; it only adds breathing room inside the list.

---

### (Optional alternative) Change 2: Remove hover translate for datacards

If you prefer *zero movement* on hover instead of adding padding, update:

```css
.vault-datacard:hover {
  transform: translateY(-1px);
}
```

to:

```css
.vault-datacard:hover {
  transform: none;
}
```

Do **not** apply this if you want to keep the “lift” effect.
