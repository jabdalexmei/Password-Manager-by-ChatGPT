## Technical Specification (EN): Information panel — render only filled fields (hide empty fields)

### Goal

In the **Information** panel (right column), display **only the fields that contain data**. If a field value is empty / whitespace / missing, the field row must not be rendered at all.

This should reduce visual noise and make the Information panel compact and readable.

### Scope

Frontend only.

### Files

* `src/features/Vault/components/Details/Details.tsx`

No backend changes. No database changes.

---

## Current behavior

`Details.tsx` currently computes `hasX` flags (e.g., `hasUrl`, `hasUsername`, `hasEmail`, etc.), but the UI still renders most `.detail-field` blocks even when the underlying value is empty (actions may hide, but the row still exists).

---

## Required behavior

### 1) Always shown

* **Title** (data card title) — always render.

### 2) Conditionally shown (only if non-empty)

Render the corresponding `.detail-field` row **only** when value is non-empty after trimming.

Fields to hide when empty:

* URL
* Username
* Email
* Password
* Mobile phone
* Note
* Folder (folder name)
* Tags

### 3) Definition of “empty”

A field is considered empty if:

* `value === null || value === undefined`, OR
* `String(value).trim().length === 0`

For tags:

* Tags are empty if `tags` is `null/undefined` or `tags.length === 0`

---

## Implementation requirements

### A) Use existing helper/flags

If `Details.tsx` already defines a helper like:

```ts
const hasValue = (v: string | null | undefined) => ...
```

or already defines flags:

```ts
const hasUrl = hasValue(card.url);
```

Reuse them (do not duplicate logic).

If a helper does **not** exist, add a local helper function in `Details.tsx`:

```ts
const hasValue = (v?: string | null) => !!v && v.trim().length > 0;
```

### B) Wrap each field block with conditional rendering

For every optional field block currently rendered as:

```tsx
<div className="detail-field"> ... </div>
```

Change to:

```tsx
{hasX && (
  <div className="detail-field"> ... </div>
)}
```

Where `hasX` matches the field.

#### Required conditions per field

* URL row: `hasUrl`
* Username row: `hasUsername`
* Email row: `hasEmail`
* Password row: `hasPassword`
* Mobile row: `hasMobilePhone`
* Note row: `hasNote`
* Folder row: `hasFolderName` where:

  * `const folderName = ...;`
  * `const hasFolderName = hasValue(folderName);`
* Tags row: `hasTags` where:

  * `const hasTags = Array.isArray(card.tags) && card.tags.length > 0;`

### C) Do not break existing actions

If a field row is shown, its existing actions must remain unchanged:

* Copy button stays available for text fields
* Password actions (copy, show/hide, generator) keep working
* URL open action stays available

When a field row is hidden, no action controls should appear for it.

---

## Acceptance Criteria

1. When viewing a card in **Information**, only the fields with non-empty values are visible.
2. Empty values do not produce “blank rows”, labels without values, or placeholder separators.
3. Title always renders.
4. Existing actions (copy/open/password controls) still work for fields that are displayed.
5. No UI regressions in layout spacing or alignment.

---

## Manual test cases

1. Card with only **Title** filled → Information shows only Title section (no other labels).
2. Card with **Title + Username + Password** → shows exactly those 3; no URL/email/note/mobile rows.
3. Card with `note = "   "` (spaces only) → Note row is hidden.
4. Card with `tags = []` → Tags row hidden; with `tags = ["a","b"]` → Tags row visible.
