

### TS (EN): Fix `open_path(..., None)` type inference (E0283)

**File:** `src-tauri/src/services/attachments_service.rs`

**Replace:**

```rust
app.opener()
    .open_path(preview_path_str, None)
    .map_err(|_| ErrorCodeString::new("ATTACHMENT_OPEN_FAILED"))
```

**With:**

```rust
app.opener()
    .open_path(preview_path_str, Option::<String>::None)
    .map_err(|_| ErrorCodeString::new("ATTACHMENT_OPEN_FAILED"))
```

(Можно и короче: `None::<String>` — но вариант с `Option::<String>::None` обычно читается лучше.)

### Acceptance criteria

* `cargo build` проходит без `E0283`.
* “Open attachment” продолжает открывать файл через дефолтное приложение ОС.
