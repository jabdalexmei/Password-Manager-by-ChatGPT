
## TS (EN): Add Tooltips namespace + fix backup icon tooltips (and ensure correct actions)

### Problem

Tooltip shows raw i18n keys (e.g. `backup.export`) instead of human text, because the translation key is missing in the current dictionary and the translation resolver returns the key when value is not found.

### Goal

1. Add a dedicated tooltip dictionary file.
2. Wire it into the app translation system (`src/lib/i18n.ts`).
3. Use that tooltip dictionary for `title` and `aria-label` on the backup Import/Export icon buttons in the Vault header.

> `aria-label` is required for icon-only buttons (accessibility). ([Aditus][2])

## Acceptance criteria

1. Hover on Export icon shows tooltip **“Export backup”**, not `backup.export`. (`title` attribute drives native tooltip.) ([Wikipedia][1])
2. Hover on Import icon shows tooltip **“Import backup”**.
3. Icon-only buttons have meaningful `aria-label` text (accessibility). ([Aditus][2])
4. Clicking Export still triggers Export flow, clicking Import triggers Import/Restore flow.
