## Technical Specification (EN): Modal UI rules — input rounding + footer buttons layout (Cancel left, Confirm right)

### Goal

Standardize modal (dialog) UI across the app:

1. **Input rounding in modals** must match the **Information** panel fields (less rounded corners).
2. **Modal footer buttons** must follow a consistent rule:

   * **Cancel** button is aligned to the **left**
   * **Confirm/Primary/Destructive** action button(s) are aligned to the **right**

### Scope

Frontend CSS only. No TS/TSX changes required (as long as button ordering remains consistent).

### Files

* `src/styles/ui.css`

---

## 1) Modal inputs rounding must match Information fields

### Problem

Inputs inside modals look “too rounded” (using `--radius-md`), while Information fields use a tighter rounding (`--radius-sm`). This creates inconsistent UI.

### Requirement

All form controls rendered inside `.dialog` must use `border-radius: var(--radius-sm)`.

### Implementation

**File:** `src/styles/ui.css`
**Add the following block** near the dialog styles section (recommended: below `.dialog-footer` block), or at the end of file.

```css
/* Dialog inputs: match Information field rounding */
.dialog .input,
.dialog .textarea,
.dialog .form-field input,
.dialog .form-field select,
.dialog .form-field textarea,
.dialog .form-textarea {
  border-radius: var(--radius-sm);
}
```

### Acceptance

* “Create folder” modal input radius matches Information fields visually.
* Same applies to Create/Edit data card modals and any other dialogs.

---

## 2) Modal footer buttons rule: Cancel left, Confirm right

### Problem

Current modal footer layout aligns all buttons to the right, so Cancel is not positioned as desired.

### Requirement

In every modal footer:

* The **Cancel** button must be aligned to the left.
* The confirmation button(s) (Create/Save/Delete/etc.) must be aligned to the right.

### Assumption / Dependency

In each modal footer, **Cancel is rendered as the first button** and uses `btn btn-secondary`.

### Implementation (CSS-only)

**File:** `src/styles/ui.css`
Add this block immediately after the `.dialog-footer` rule:

```css
/* Dialog footer: keep Cancel on the left, actions on the right */
.dialog-footer > .btn.btn-secondary:first-child {
  margin-right: auto;
}
```

This pushes the first secondary button (Cancel) to the left, while keeping the rest aligned right.

### Acceptance

* In “Create folder” modal:

  * Cancel is on the left
  * Create is on the right
* Same rule applies to all dialogs using `.dialog-footer`.

2. Check Create/Edit data card dialogs: footer buttons still aligned correctly.
3. Check ConfirmDialog: Cancel left, Confirm/Delete right.
4. Ensure dialogs with only one button remain aligned correctly (no weird spacing).
