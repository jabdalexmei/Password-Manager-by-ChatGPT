
## Technical Specification (EN): Vault “All items” list visual fixes (remove list stroke, move Favorite badge, unify column backgrounds)

### Goal

Update Vault UI to match the “After” design:

1. All 3 columns (Navigation / All items / Information) use the **same background** as the Navigation column.
2. In **All items**, remove the **outer stroke** (border) around the list container (no “card-in-card” look).
3. In each data card row, “Favorite” is rendered as a **right-side badge** and does **not** break or join the meta text line.

### Scope

Frontend only:

* `src/styles/screens/vault.css`
* `src/features/Vault/components/DataCards/DataCards.tsx`

No backend changes.

---

## 3) Move “Favorite” badge to the right in each card (do not break meta line)

### 3.1 Update markup in DataCards list item

### File

`src/features/Vault/components/DataCards/DataCards.tsx`

### Current snippet (inside `cards.map`)

```tsx
<div className="datacard-title">{card.title}</div>
<div className="datacard-meta">
  <span>{meta}</span>
  <div>
    {isFavorite && <span className="pill">{t('label.favorite')}</span>}
    <span className="muted">{updatedText}</span>
  </div>
</div>
```

### Replace with (Favorite becomes right-side badge)

```tsx
<div className="datacard-top">
  <div className="datacard-title">{card.title}</div>
  {isFavorite && <span className="pill datacard-favorite">{t('label.favorite')}</span>}
</div>

<div className="datacard-meta">
  <span>{meta}</span>
  <span className="muted">{updatedText}</span>
</div>
```

### 3.2 Add CSS for new layout

### File

`src/styles/screens/vault.css`

Add **below `.datacard-title`** (or near datacard styles):

```css
.datacard-top {
  display: flex;
  align-items: flex-start;
  justify-content: space-between;
  gap: 12px;
}

.datacard-favorite {
  flex: 0 0 auto;
  white-space: nowrap;
}
```

Also update `.datacard-meta` to prevent weird wrapping caused by nested div:
Find:

```css
.datacard-meta {
  display: flex;
  flex-wrap: wrap;
  gap: 8px;
  align-items: center;
  color: var(--color-text-muted);
  font-size: 12px;
  line-height: 16px;
}
```

Replace with (single clean line, same look as “After”):

```css
.datacard-meta {
  display: flex;
  flex-wrap: nowrap;
  gap: 10px;
  align-items: center;
  color: var(--color-text-muted);
  font-size: 12px;
  line-height: 16px;
}
```

(If later you need fallback wrapping on very narrow widths, use `flex-wrap: wrap;` but keep the new markup.)

### Acceptance

* When a card is favorite, the “Favorite” pill is rendered **on the right**, aligned with the title row.
* The meta line is always a clean `"No meta   Updated: ..."` line and never contains “Favorite” in-between.

---

## Final Acceptance Criteria (visual)

1. The list container in **All items** has **no outer border/stroke**.
2. “Favorite” pill is on the right (title row), not inside meta.
3. Column backgrounds are unified (same as Navigation).
4. Selected/active card outline still works (blue stroke on the card itself).
