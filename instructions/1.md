

## Technical Specification (TS): Bank card Details — compact Expiry + CVC row (no full-width stretch)

### Problem

`BankCardDetails` renders Expiry and CVC inside `.bankcard-detail-grid`. 
Current CSS uses grid with a `1fr` column, causing the CVC field to expand and look visually incorrect. 

### Goal

* Keep Expiry + CVC on the **same row** (desktop).
* Make both fields **compact** (fixed / capped width), matching the mock screenshot.
* On small screens, allow stacking (expiry above cvc).

### Non-goals

* No DB/backend changes
* No changes to create/edit dialogs
* No changes to other detail fields (Title/Number/Note/etc.)

---

## Implementation

### A) Update layout for `.bankcard-detail-grid` from CSS Grid → Flex row (compact)

**File:** `src/styles/screens/vault.css` 

#### 1) Replace the existing block

Find the current block:

```css
.bankcard-detail-grid {
  display: grid;
  grid-template-columns: minmax(160px, 220px) minmax(240px, 1fr);
  gap: 12px;
  align-items: start;
}

@media (max-width: 900px) {
  .bankcard-detail-grid {
    grid-template-columns: 1fr;
  }
}
```

#### 2) Replace it with the following (exact)

```css
/* Bank card details: Expiry + CVC should be compact (match mock) */
.bankcard-detail-grid {
  display: flex;
  gap: 12px;
  align-items: flex-start;
  flex-wrap: wrap; /* allows stacking on narrow widths */
}

/* prevent "stretch to full width" behavior */
.bankcard-detail-grid > .detail-field {
  flex: 0 0 auto;
  min-width: 0;
}

/* compact widths to match mock screenshot */
.bankcard-detail-grid > .detail-field:first-child {
  width: 180px; /* Expiry date */
}

.bankcard-detail-grid > .detail-field:last-child {
  width: 260px; /* CVC/CVV + action buttons */
}

/* value box should fill the compact field width */
.bankcard-detail-grid .detail-value-box {
  width: 100%;
}

/* Mobile: stack vertically */
@media (max-width: 900px) {
  .bankcard-detail-grid > .detail-field:first-child,
  .bankcard-detail-grid > .detail-field:last-child {
    width: 100%;
  }
}
```

**Rationale**

* Flex row + `flex: 0 0 auto` prevents the second field from consuming remaining space (unlike `1fr` in grid).
* Fixed widths give the exact “small boxes” look.
* `flex-wrap: wrap` keeps it robust on small windows.

---

## Acceptance Criteria

1. In Bank card **Details panel**, Expiry date and CVC/CVV appear on the **same row** on desktop.
2. Expiry field width is compact (~180px).
3. CVC field width is compact (~260px) and still fits the copy/eye buttons without wrapping.
4. On narrow window widths (<900px), Expiry and CVC stack vertically (full width).
5. No visual changes to Title/Holder/Number/Note/Tags fields.
