

## Technical Specification (TS): Bank card Details — Expiry & CVC as normal vertical fields, but compact width

### Goal

In **Bank Card Details** panel:

1. `Expiry date` and `CVC/CVV` are rendered as **separate vertical fields** (one under another), same as other fields.
2. Their `.detail-value-box` width is **shorter** (capped), but height/visual style matches other fields.
3. Copy/Eye buttons stay inside the CVC value box and layout remains stable.

### Files

* `src/features/Vault/components/BankCards/BankCardDetails.tsx`
* `src/styles/screens/vault.css`

---

## 1) Frontend: remove “grid row” wrapper and render as normal fields

### File: `src/features/Vault/components/BankCards/BankCardDetails.tsx`

#### Replace this block (exactly)

Find:

```tsx
<div className="bankcard-detail-grid">
  <div className="detail-field">
    ...
  </div>

  <div className="detail-field">
    ...
  </div>
</div>
```

#### Replace with (exactly)

```tsx
<div className="detail-field bankcard-compact-expiry">
  <div className="detail-label">{t('label.expiry')}</div>
  <div className="detail-value-box">
    <div className="detail-value-text">{card.expiryMmYy || tCommon('value.empty')}</div>

<div className="detail-field bankcard-compact-cvc">
  <div className="detail-label">{t('label.cvc')}</div>
  <div className="detail-value-box">
    <div className="detail-value-text detail-value-text-monospace">{cvcDisplay}</div>
    <div className="detail-value-actions">
      <button
        className="icon-button"
        type="button"
        aria-label={t('action.copy')}
        onClick={() => detailActions.copyToClipboard(card.cvc, { isSecret: true })}
      >
        <IconCopy />
      </button>
      <button
        className="icon-button"
        type="button"
        aria-label={detailActions.showCvc ? t('action.hide') : t('action.reveal')}
        onClick={detailActions.toggleCvcVisibility}
      >
        {detailActions.showCvc ? <IconPreviewOff /> : <IconPreview />}
      </button>
    </div>
  </div>
</div>
```

## 2) CSS: delete grid styles and add compact width caps

### File: `src/styles/screens/vault.css`

#### A) Remove the grid rules

Delete this entire block:

```css
.bankcard-detail-grid {
  display: grid;
  grid-template-columns: minmax(160px, 220px) minmax(240px, 1fr);
  gap: 12px;
  align-items: start;
}

@media (max-width: 900px) {
  .bankcard-detail-grid {
    grid-template-columns: 1fr;
  }
}
```

#### B) Add compact width rules (append near bankcard section)

```css
/* Bank card details: Expiry/CVC should be normal vertical fields, but shorter width */
.bankcard-detail-card .bankcard-compact-expiry .detail-value-box {
  max-width: 520px;
}

.bankcard-detail-card .bankcard-compact-cvc .detail-value-box {
  max-width: 360px;
}
```

Notes:

* These numbers (`520px` / `360px`) are chosen to match the “shorter but not tiny” look and fit two icon buttons.
* If you want them even shorter: reduce them (e.g. `480px / 320px`).

`max-width` is the correct tool here because it prevents the element from becoming wider than the cap. ([MDN Web Docs][1])

## Acceptance Criteria

1. `Expiry date` is **NOT** on the same row with `CVC/CVV` — it is above it.
2. Both fields visually match the standard `.detail-field` style.
3. Their value boxes are **shorter** than the full-width fields, matching your screenshot intent.
4. CVC copy/reveal buttons remain visible and aligned inside the CVC box.
